# プロジェクト構成変更ログ

## 変更日
2026年1月15日

## 変更内容
開発を効率化するため、フラットなファイル構成からディレクトリ分けされた構成へ変更しました。

### 変更前
```text
/
├── main.py (or main)
├── readme.md
├── AI案.md
└── colabo.md
```

### 変更後
```text
/
├── src/                # ソースコード用
│   └── main.py         # ゲームエンジン及びAI実装
├── doc/                # ドキュメント用
│   ├── specification.md # 仕様書 (旧 readme.md)
│   ├── strategy.md      # AI戦略案 (旧 AI案.md)
│   ├── logs/           # 開発ログ・ナレッジ
│   └── misc/           # その他資料
│       └── colab_notebook.md (旧 colabo.md)
```

## 目的
* **src**: 実行可能なコードを分離し、誤ってドキュメントを編集するリスクを減らす。
* **doc**: 仕様や戦略メモを一箇所にまとめ、情報の検索性を高める。
* **logs**: 試行錯誤の過程や、得られた知見を時系列で保存する。
